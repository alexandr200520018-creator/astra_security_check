# 📋 Инструкция по использованию скрипта проверки безопасности Astra Linux

## 🔒 Security Audit Tool для Astra Linux

Инструмент для комплексной проверки параметров безопасности операционной системы Astra Linux Special Edition в соответствии с требованиями ФСТЭК. Скрипт анализирует настройки ядра, политики паролей, конфигурацию SSH, инструменты безопасности Astra Linux и информацию об учётных записях пользователей.

## ⚙️ Требования

- **Операционная система**: Astra Linux Special Edition 1.7.x
- **Права доступа**: root (запуск через `sudo`)
- **Зависимости**: Стандартные утилиты Linux (все присутствуют в базовой установке Astra Linux):
  - `bash` (4.0+)
  - `grep`, `awk`, `sed`, `cut`
  - `sysctl`, `systemctl`
  - `sudo` для доступа к защищённым файлам

Установка дополнительных пакетов **не требуется**.

## 🚀 Быстрый старт

### 1. Скачайте скрипт

```bash
wget wget https://raw.githubusercontent.com/alexandr200520018-creator/astra_security_check/main/astra_security_check.sh
# или скопируйте содержимое скрипта в файл
```

### 2. Дайте права на выполнение

```bash
chmod +x astra_security_check.sh
```

### 3. Запустите проверку

```bash
sudo ./astra_security_check.sh
```

### 4. Получите результат

Скрипт создаст файл отчёта в текущей директории:
```
security_report_full_YYYYMMDD_HHMMSS.txt
```

## 📊 Структура отчёта

Отчёт содержит **5 разделов**, полностью соответствующих требованиям ФСТЭК:

### Раздел 1: Парольная политика
Проверяет параметры аутентификации и сложности паролей:

| Параметр | Файл конфигурации | Проверяемое значение |
|----------|-------------------|----------------------|
| Максимальный срок действия пароля | `/etc/login.defs` | PASS_MAX_DAYS |
| Минимальный интервал смены пароля | `/etc/login.defs` | PASS_MIN_DAYS |
| Предупреждение об истечении пароля | `/etc/login.defs` | PASS_WARN_AGE |
| Количество неудачных попыток входа | `/etc/pam.d/common-auth` | deny |
| Требования к сложности пароля | `/etc/pam.d/common-password` | minlen, dcredit, ucredit, lcredit, ocredit |
| История паролей | `/etc/pam.d/common-password` | remember |
| Блокировка учётной записи | `/etc/pam.d/common-auth` | lock_time, unlock_time |

### Раздел 2: Инструменты командной строки Astra Linux
Проверяет статус всех инструментов безопасности Astra Linux:

| Инструмент | Описание |
|------------|----------|
| `astra-sudo-control` | Запрос пароля при sudo |
| `astra-shutdown-lock` | Блокировка выключения/перезагрузки |
| `astra-mount-lock` | Запрет монтирования носителей |
| `astra-format-lock` | Запрет форматирования |
| `astra-mic-control` | Мандатный контроль целостности |
| `astra-ufw-control` | Межсетевой экран |
| `astra-ptrace-lock` | Блокировка трассировки ptrace |
| `astra-lkrg-control` | Модуль ядра LKRG |
| `astra-rootloginssh-control` | Ограничение root по SSH |
| `astra-docker-isolation` | Изоляция Docker |
| `astra-digsig-control` | Контроль целостности ELF-файлов |
| `astra-secdel-control` | Очистка памяти |
| `astra-swapwiper-control` | Очистка разделов подкачки |

**Всего проверяется 24 инструмента безопасности**.

### Раздел 3: Параметры безопасности ядра
Проверяет системные параметры через `sysctl`:

| Параметр ядра | Описание |
|---------------|----------|
| `net.ipv4.ip_forward` | IP-переадресация |
| `net.ipv4.conf.all.accept_redirects` | ICMP-перенаправления |
| `fs.protected_hardlinks` | Защита жёстких ссылок |
| `fs.protected_symlinks` | Защита символических ссылок |
| `fs.suid_dumpable` | Core dump для setuid-файлов |
| `kernel.randomize_va_space` | ASLR (рандомизация адресного пространства) |
| `kernel.dmesg_restrict` | Доступ к журналу ядра |
| `kernel.yama.ptrace_scope` | Ограничение ptrace |
| `kernel.perf_event_paranoid` | Доступ к событиям производительности |

**Всего проверяется 12 параметров ядра**.

### Раздел 4: Конфигурация SSH сервера
Проверяет настройки `/etc/ssh/sshd_config`:

| Параметр | Проверяемое значение |
|----------|---------------------|
| `Port` | Порт SSH |
| `ListenAddress` | Адреса прослушивания |
| `LoginGraceTime` | Время ожидания входа |
| `PermitRootLogin` | Разрешение входа root |
| `StrictModes` | Проверка прав файлов |
| `MaxAuthTries` | Максимум попыток аутентификации |
| `MaxSessions` | Максимум сессий |
| `PubkeyAuthentication` | Аутентификация по ключам |
| `fail2ban` | Наличие и статус fail2ban |

### Раздел 5: Информация об учётных записях пользователей
Анализирует всех пользователей системы (UID ≥ 1000) и root:

| Параметр | Источник |
|----------|----------|
| Статус учётной записи | `/etc/shadow` |
| Парольная информация | `/etc/shadow` (заблокирован/зашифрован) |
| Дни после последней смены пароля | `/etc/shadow` (поле 3) |
| Минимальные дни до смены | `/etc/shadow` (поле 4) |
| Максимальный срок действия | `/etc/shadow` (поле 5) |
| Дни предупреждения | `/etc/shadow` (поле 6) |
| Дни неактивности до блокировки | `/etc/shadow` (поле 7) |
| Срок действия учётной записи | `/etc/shadow` (поле 8) |
